# 脱敏与公开展示规则

## 基本原则

本项目来自真实业务场景的方法抽象，但 GitHub 公开版本只展示产品方法、流程结构和通用案例，不展示任何可识别真实项目的信息。

## 不公开的内容

以下内容不进入公开仓库：

- 前司名称
- 真实项目名称
- 真实剧场名称
- 真实角色/NPC 名称
- 剧情文本、人物关系和世界观设定
- 票务规则、价格策略、内部运营口径
- 真实用户聊天记录
- 真实 Dify 知识库内容
- 未脱敏的 Prompt 原文
- 未脱敏的 Dify 工作流导出文件
- 内部数据、订单数据、用户数据或测试数据

## 可以公开的内容

以下内容可以在脱敏后公开：

- 通用产品背景和业务问题
- 用户流程图
- 脱敏原型图
- 通用工作流结构
- Prompt 设计思路
- 测试方法和 Badcase 分类
- 虚构示例问答
- 不包含真实业务词的 Dify 工作流截图

## 命名规则

公开版本统一使用通用表达：

- `AI 剧场客服管家`
- `线下体验服务`
- `体验前推荐`
- `体验后复盘`
- `通用知识库`
- `人工接管`

避免出现任何能反向定位真实项目的信息。

## 截图处理要求

上传任何截图前，需要检查：

- 是否包含真实项目名称
- 是否包含真实角色或剧情词
- 是否包含公司、团队、账号、知识库名称
- 是否包含用户信息、订单信息、联系方式
- 是否包含后台链接、API Key、Token 或内部路径

如无法确认，优先不上传截图，改用重绘流程图或文字说明。

## 面试与公开展示的区别

公开展示需要强脱敏。面试时可以适度说明项目来自真实业务场景，但仍应避免泄露真实剧情、角色、用户数据和内部运营资料。
