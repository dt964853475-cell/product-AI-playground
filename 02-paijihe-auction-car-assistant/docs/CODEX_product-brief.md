# 拍即合：法拍车出价助手产品说明

## 项目定位

拍即合是一个面向法拍车新手用户的小程序 MVP。它的目标不是替用户做最终交易决策，而是把参拍前最分散、最容易漏看的信息整理到一条流程里，帮助用户更快完成车辆信息查询、车况判断、估值参考和总成本测算。

公开版本只展示产品设计、流程拆解和能力规划。所有车辆、价格、维保、理赔和成本数据均为演示数据，不构成任何真实交易建议。

## 为什么做这个项目

法拍车对新手用户并不友好。用户通常需要同时理解法院公告、车辆基础信息、维保记录、理赔记录、二手车市场价格、跨区域提车成本、过户和整备成本等信息。

真正麻烦的地方不只是“查不到信息”，而是信息太分散。用户需要在多个平台之间来回切换，再自己判断这些信息对出价有没有影响。对于第一次参拍的人来说，这个过程很容易漏项，也很难形成一个清晰的最高出价边界。

所以这个 MVP 先聚焦一个具体问题：

> 用户看到一辆法拍车后，能不能在几分钟内得到一份结构化的参拍参考？

## 目标用户

- 第一次接触法拍车、不了解参拍流程的新手用户。
- 想快速判断某辆车是否值得继续研究的潜在参拍用户。
- 需要把车辆信息、市场价格和办理成本放在一起看的用户。
- 希望用一个轻量工具完成初步估算，而不是一开始就投入大量人工查询成本的用户。

## 核心问题

| 问题 | 用户表现 | 产品应对 |
| --- | --- | --- |
| 信息分散 | 用户要到不同平台查车辆、估值、维保、理赔和地图距离 | 通过 VIN 输入和流程化页面串联查询链路 |
| 成本不透明 | 用户只看到成交价，却忽略过户、拖车、整备、差旅等额外成本 | 将成本拆成“车辆本身成本”和“拍得者额外成本” |
| 出价缺少边界 | 用户不知道市场价和总成本之间还有多少利润空间 | 给出估值区间、总落地成本和建议最高出价 |
| 风险认知不足 | 新手容易把估算结果当成确定结论 | 在页面中加入风险提示和免责声明 |
| 数据无法沉淀 | 用户真实办理成本没有回流，下一次估算无法变准 | 设计实际成本上传和反馈次数机制 |

## MVP 功能范围

当前 MVP 聚焦六个核心功能：

1. VIN 输入与基础车辆信息识别。
2. 车况、维保记录、理赔记录展示。
3. 二手车市场估值和参考车源展示。
4. 三地输入，用于估算跨区域办理成本。
5. 总落地成本测算和建议最高出价。
6. 用户上传实际成本，用于后续优化估算模型。

## 用户价值

拍即合不替代专业检测、法律咨询或最终交易判断。它更像一个参拍前的“计算草稿本”，把用户原本分散在脑子里、表格里和多个平台里的信息先整理出来。

对新手来说，它的价值在于三点：

- 看清楚这辆车大概值多少钱。
- 看清楚自己拍下来以后还要花多少钱。
- 在出价前知道自己应该保守到什么位置。
